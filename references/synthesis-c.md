# GRAFCET → C 高階合成（MIAT 階段三：軟體合成）

## 合成總則

- 每個 GRAFCET **狀態＝一個全域布林變數（狀態暫存器）** `X0, X1, …`：值 1＝活躍、0＝非活躍（one-hot）。
- 每個**轉移＝一個 if 陳述式**：條件＝「來源狀態活躍 AND 轉移條件」；本體＝「來源狀態設 0、目標狀態設 1」。
- **控制器（Controller）與資料路徑（Datapath）分離**：控制器由下列規則機械式合成、保證正確；Datapath（動作函式）由工程師手寫或 LLM 生成。
- 合成出的系統程式可取代作業系統，直接控制底層硬體。

## 固定程式架構（三層，不可改）

```c
void main(void) {
    while (1) {
        grafcet();          /* 每次呼叫＝一個掃描週期（等同一個時脈） */
    }
}

void grafcet(void) {
    /* (1) 輸入取樣：轉移條件所需的感測器/按鈕訊號，每掃描週期開頭更新一次 */
    /* (2) 狀態轉移 if 鏈（依下方五大規則合成） */
    action();               /* (3) 轉移判斷完才執行動作 */
}

void action(void) {
    if (X1 == 1) { /* 狀態1的動作 */ }
    if (X2 == 1) { /* 狀態2的動作 */ }
    /* … 每個有動作的狀態一個 if … */
}
```

補充規範：

- **輸入取樣**：轉移條件引用的外部訊號（感測器、按鈕）在 `grafcet()` 開頭每週期取樣一次到全域變數，等同同步系統的時脈取樣語義。
- **if 與 else-if 的選擇**：單一 GRAFCET 內任何時刻只有一個狀態活躍時，轉移鏈可用 `else if` 串接（課程範例作法，較有效率）；但**含平行（AND）結構的模型，轉移與 `action()` 的分派都必須用獨立 `if`**，否則同一週期多個並行狀態的轉移/動作會被 `else if` 略過。Convergence OR 的各分支一律用獨立 `if`（見規則 5）。
- **狀態停留**：若目前狀態所有離開條件皆不成立，狀態自然停留、動作持續執行——這是合法且常用的等待語義，對應 GRAFCET「禁止自迴圈、停留即可」的規定。
- **命名**：狀態暫存器 `X0` 或 `x0` 皆可（課程教材兩者並見），同一專案內保持一致。

## 五大基本結構合成規則

### 規則 1：Fundamental Block（基本轉移）

```c
if ((X0 == 1) && (R0 == 1)) {
    X0 = 0;
    X1 = 1;
}
```

### 規則 2：Divergence AND（並行分歧，雙線）— 同時激活所有後續狀態

```c
if ((X0 == 1) && (R0 == 1)) {
    X0 = 0;
    X1 = 1;
    Xn = 1;      /* 所有並行分支同時設 1 */
}
```

### 規則 3：Convergence AND（並行匯聚，雙線）— 全部並行狀態到齊才轉移

```c
if ((X0 == 1) && (X1 == 1) && (R0 == 1)) {
    X0 = 0;
    X1 = 0;      /* 所有並行狀態同時清 0 */
    Xn = 1;
}
```

### 規則 4：Divergence OR（分支分歧，單線）— if / else if，條件互斥

```c
if ((X0 == 1) && (R0 == 1)) {
    X0 = 0;
    X1 = 1;
}
else if ((X0 == 1) && (R1 == 1)) {
    X0 = 0;
    X2 = 1;
}
```

### 規則 5：Convergence OR（分支匯聚，單線）— 各分支獨立 if，指向同一目標

```c
if ((X0 == 1) && (R0 == 1)) {
    X0 = 0;
    Xn = 1;
}
if ((X1 == 1) && (R1 == 1)) {
    X1 = 0;
    Xn = 1;
}
```

## 補充規則

### Sub-Grafcet → 函式呼叫

子 GRAFCET 合成為獨立函式（子控制器）。父狀態的動作＝呼叫子控制器函式：

```c
void action(void) {
    if (x1 == 1) grafcet1();     /* 動作是子系統 → 呼叫子控制器 */
    if (x2 == 1) preEmphasis();  /* 動作是單純函式 → 直接呼叫 Datapath */
}
```

### 父子控制器握手（樣板，必須照抄結構）

子控制器結束狀態暫存器（如 `x14`）＝父控制器轉移條件；轉移時**同時清除子結束狀態、復位子初始狀態**：

```c
/* 父控制器：狀態1（執行子系統grafcet1）→ 狀態2 */
else if (x1 == 1 && x14 == 1) {
    x1  = 0;      /* 離開父狀態 */
    x14 = 0;      /* 清除子結束旗標 */
    x10 = 1;      /* 子控制器復位回初始狀態 */
    x2  = 1;      /* 進入父的下一狀態 */
}
```

子控制器的啟動：父狀態暫存器（`X1==1`）直接作為子初始狀態（10→11）的轉移條件。**全程只用全域狀態暫存器溝通，不設計任何額外通訊協定。**

### Reset

```c
void rst(void) {
    X0 = 1;                      /* 只有初始狀態為 1 */
    X1 = X2 = X3 = X4 = 0;       /* 其餘狀態全 0 */
    /* 輸出與旗標一併歸零 */
}
```

觸發方式二擇一：上電/硬體重置後於 `main()` 開頭呼叫 `rst()`；或每週期輪詢 RESET 訊號，為高時呼叫 `rst()`（對應 VHDL 版 `IF RST='1'` 的語義）。

### 計時器（時間條件）

進入狀態後於動作中累加計時變數，轉移條件寫計時到值：動作 `timeRY++;`，條件 `timeRY == 5`。轉移時將計時變數歸零。

## 完整範例：語者辨識系統主控制器

IDEF0：A1 音訊擷取（子系統）、A2 預強調、A3 特徵擷取（子系統，內含 A33 LPCC 子子系統）、A4 端點偵測、A5 分類。
GRAFCET0 狀態 0–5；GRAFCET1 狀態 10–15；GRAFCET3 狀態 30–35；GRAFCET34 狀態 340–344。

```c
void grafcet0(void) {
    if (x0 == 1 && soundSource == 1) {        /* 0→1 */
        x0 = 0;  x1 = 1;
    }
    else if (x1 == 1 && x14 == 1) {           /* 1→2：子grafcet1完成 */
        x1 = 0;  x14 = 0;  x10 = 1;  x2 = 1;
    }
    else if (x2 == 1) {                       /* 2→3：永真(=1) */
        x2 = 0;  x3 = 1;
    }
    else if (x3 == 1 && x35 == 1) {           /* 3→4：子grafcet3完成 */
        x3 = 0;  x35 = 0;  x30 = 1;  x4 = 1;
    }
    else if (x4 == 1) {                       /* 4→5 */
        x4 = 0;  x5 = 1;
    }
    else if (x5 == 1) {                       /* 5→0：回初始 */
        x5 = 0;  x0 = 1;
    }
    action();
}

void action(void) {
    if      (x1 == 1) grafcet1();            /* 音訊擷取子控制器 */
    else if (x2 == 1) preEmphasis();         /* Datapath：LLM 可生成 */
    else if (x3 == 1) grafcet3();            /* 特徵擷取子控制器 */
    else if (x4 == 1) endpointDetection();
    else if (x5 == 1) classification();
}
```

對應的狀態/動作說明表（建模文件必附；節錄）：

| 狀態 | 動作 | 函式名稱 | 功能說明 |
|---|---|---|---|
| 0 | Null | Null | 初始狀態，等待聲音訊號 |
| 1 | 音訊擷取 | grafcet1() | 按鈕後儲存麥克風聲音資料 |
| 2 | 預強調 | preEmphasis() | 高通濾波增強高頻 |
| 3 | 特徵擷取 | grafcet3() | 對每個音框擷取特徵 |
| 14 | Null | Null | 子系統結束旗標狀態 |
| 341 | AutoCorrelation | autoCorrelation() | 取得自相關係數 |

## Datapath / Driver 的 LLM 協同生成

動作函式（演算法、驅動）交給 LLM 生成。Prompt 要明確給：輸入參數（取樣率、訊號長度、陣列）、輸出參數、要含的處理步驟與演算法（如 Levinson-Durbin）。要求**最簡單、易維護的語法**，可要求每個函式不超過十行，便於除錯。控制器一律由合成規則產生，**不交給 LLM 自由發揮**。

## 驗證（步驟 6）

1. **軟體模擬狀態轉移**：觀察狀態暫存器序列是否符合模型預期（例：x0=1 → 按鈕 → x1=1 且 x10→x11→x12⇄x13→x14 → x2=1 → …）。
2. 記錄 **code size、記憶體使用量、效能分析** 三項指標。
3. 程式與模型一對一對應：測試失敗可直接回溯到對應狀態或轉移，改模型後重新合成。
