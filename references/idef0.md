# IDEF0 系統架構設計（MIAT 階段一）

## 兩種架構 — 必須分開繪製

設計開始時，先區分並**分別獨立繪製**兩張圖：

| | 實體架構 Physical Architecture | 功能架構 Functional Architecture |
|---|---|---|
| 比喻 | 系統的「肉體」 | 系統的「靈魂」 |
| 回答 | 由哪些硬體/軟體實體構成？分佈在哪些節點？用什麼介面連接？ | 有哪些功能與子功能？如何分工協作？輸入-處理-輸出的邏輯流向？ |
| 內容 | MCU、感測器、馬達、通訊晶片、節點（中央單元 Gateway / 周邊單元 End Device）、通訊協定（UART/CAN/SPI/I2C/BLE/Wi-Fi） | 功能模組、模組間資料流（與硬體部署無關） |
| 性質 | 靜態、物理部署；影響 BOM 與成本 | 動態、邏輯資訊流；貫穿整個開發生命週期 |

把軟體功能與硬體模組混為一談，是開發後期整合問題的常見根源。

## IDEF0 語言規範

IDEF0（Integrated Definition for Function Modeling，源自美國空軍 1970 年代 ICAM 計畫）以嚴謹的圖形規範彌補自然語言描述的模糊。

### 元素

- **方塊（Activity Box）**：代表一個活動/功能，名稱必須是**動詞或動詞片語**（「處理資料」「擷取影像」「傳送訊息」）。
- **箭頭（Arrow）**：代表流動的資料或物件，名稱必須是**名詞**（「串流影像」「控制指令」「歸屬值」）。

### ICOM 四方向箭頭

```
                 Control 控制（上方進入）
                 限制條件/規範/參數，如標準、交期、價格表
                        │
                        ▼
   Input 輸入 ──►┌──────────────┐──► Output 輸出
   執行功能所需   │  動詞命名的   │    功能執行後的
   的資料(原料)   │  功能方塊     │    結果(成品)
                 └──────────────┘
                        ▲
                        │
                 Mechanism 機制（下方進入）
                 執行資源：人員/工具/設備/軟體庫/AI工具
```

- 嵌入式系統實作最常用 **輸入、輸出、活動** 三者；Control/Mechanism 多用於專案管理層（資源規劃、風險管理）。
- **一個功能的輸出可以是下一個功能的輸入**，串聯形成系統流程。

## 階層式分解規則

1. 頂層一個方塊 **A0** 代表整個系統（含對外輸入/輸出）。
2. A0 分解為 A1, A2, … An（課程範例常見 2–5 個子系統）。
3. 逐層編號：A3 → A31, A32, …；A33 → A331, A332, …。編號即階層路徑。
4. **分解終止條件：原子模組（Atomic Module）**——功能足夠單一、小到 AI（LLM）能精確生成其程式碼的粒度（例如一個 FFT、一個影像處理演算法）。不必無限分解。
5. 每個模組必須有清晰的職責與介面（輸入/輸出明確標示），使團隊成員只依介面即可獨立開發、無縫整合。

## 與後續階段的銜接

- 每個 IDEF0 功能模組被視為一個獨立的**離散事件系統（DES）**，用一張 GRAFCET 建模其行為（見 grafcet.md）。
- IDEF0 描述系統的「是什麼」（結構，類比資料結構）；GRAFCET 描述「如何做」（行為，類比演算法）。
- 模組間箭頭 → 合成階段的全域變數/訊號介面。

## FBD 功能方塊圖定義（工作流程步驟 3）

IDEF0 架構完成後、GRAFCET 建模之前，為每個功能模組詳細定義輸入、輸出與控制流程。標準產出是介面定義表（每模組一列）：

| 模組 | 函式/介面 | 輸入 | 輸出 | 控制/參數 |
|---|---|---|---|---|
| A1 擷取循跡訊號 | readIR(&SL,&SR,threshold) | ADC 原始值 | SL、SR 旗標 | 判別門檻值 |

此表直接對應 GRAFCET 的轉移條件（輸入）與動作（輸出），以及合成階段的函式簽名。

## 課程範例

### 模糊控制器晶片（經典範例）

- A0 模糊控制器：輸入=精確數值、輸出=控制數值
- 第一層：A1 模糊化（輸出「歸屬值」）→ A2 模糊推論（輸出「推論輸出」）→ A3 解模糊化
- 第二層：A2 → A21 累加器（輸出「累加值」）→ A22 除法器

### TPRAI 教育機器人

- A1 影音感測（A11 影像擷取、A12 語音擷取）
- A2 WIFI 通訊（A21 影像資料處理/Web伺服器、A22 遠端應用程式）
- A3 影音辨識（A31 深度學習應用模組：A311 模型載入、A312 網頁應用介面；A32 語音關鍵字辨識：A321 預強調、A322 音框擷取、A323 漢明窗、A324 MFCC 特徵擷取）
- A4 機器人控制（A41 接收控制指令、A42 運動控制/PWM）

### 語者辨識系統

- A1 音訊擷取（A11 ADC、A12 儲存待辨識音訊）
- A2 預強調（簡單，直接實作為函式）
- A3 特徵擷取（A33 LPCC → A331 自相關、A332 LPC/Levinson-Durbin、A333 倒頻譜）
- A4 端點偵測、A5 分類（簡單模組直接實作為函式）

### Lidar 避障自走車

- A1 感測（A11 距離感測 VL53L0X ToF、A12 姿態感測 MPU-9250）
- A2 決策（A21 ToF-Lidar 掃描）
- A3 運動控制
