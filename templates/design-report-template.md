# MIAT 系統設計報告模板

> 依 MIAT 方法論八步驟撰寫；每一節都是必要產出物。若系統無軟體模組，可略過第 5、6 節直接到第 7 節。

## 1. 功能模組原理（演算法）說明

- 系統目標與需求
- 核心演算法原理（文字＋公式/流程說明）
- 輸入/輸出定義

## 2. 系統架構設計

### 2.1 實體架構圖（Physical Architecture）

- 硬體元件（MCU、感測器、致動器、通訊晶片）
- 部署節點（中央單元/周邊單元）
- 通訊連線與協定（UART/CAN/SPI/I2C/BLE/Wi-Fi…）

### 2.2 IDEF0 功能架構圖（Functional Architecture）

- A0 頂層圖（系統整體輸入/輸出）
- 逐層分解圖（A1, A2, …；必要時 A31, A32, …）
- 方塊＝動詞命名；箭頭＝名詞命名；分解至原子模組為止

## 3. 功能方塊圖（FBD）定義

每個功能模組列出：

| 模組 | 輸入 | 輸出 | 控制/參數 |
|---|---|---|---|
| A1 … | | | |

## 4. 離散事件建模（GRAFCET）

- 每個功能模組一張 GRAFCET 圖（建議以 Visio 或等效工具繪製）
- 階層編號：狀態 N 的子模型從 N0 起編
- **狀態/動作說明表**（必附）：

| 狀態 | 動作 | 函式名稱 | 功能說明 |
|---|---|---|---|
| 0 | Null | Null | 初始狀態，等待… |

## 5. 系統程式合成

- 依五大合成規則產生控制器（`main()` / `grafcet()` / `action()` 三層）
- 附完整程式碼；標註每個 if 對應的轉移

## 6. 系統程式驗證

- 狀態暫存器轉移序列模擬結果（與模型比對）
- Code size：____；記憶體使用量：____；效能分析：____

## 7. Driver 程式設計 / AI 生成

- 各周邊驅動函式清單與來源（手寫 / LLM 生成，附 prompt）

## 8. 軟硬體整合驗證

- 測試項目與結果（控制器軟體＋Driver＋硬體平台）
- 問題與回溯（失敗時回溯到對應狀態/轉移，改模型重新合成）
